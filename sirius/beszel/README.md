# Beszel

Server monitoring hub and agent. The hub runs on Sirius and provides 
the web UI. Agents run on each monitored host.

## Access

- **UI:** beszel.aeoncore.net
- **Internal port:** 8090

## Architecture

Beszel uses a hub + agent model. The hub (this stack) runs on Sirius.
Separate agents run on Deneb (and any other hosts to be monitored).
The Sirius agent communicates with the hub via a shared Unix socket 
rather than over the network.

## Notes

- The agent uses `network_mode: host` to monitor host-level metrics
- Communication between hub and agent is via Unix socket at 
  `/beszel_socket/beszel.sock`
- Docker socket is mounted read-only for container monitoring
- The SSH key and token in `.env` are generated by Beszel on first run —
  retrieve them from the Beszel UI after initial startup

## Volumes

| Host path | Container path | Purpose |
|-----------|----------------|---------|
| /srv/appdata/beszel/beszel_data | /beszel_data | Hub database and config |
| /srv/appdata/beszel/beszel_socket | /beszel_socket | Unix socket (shared with agent) |
| /srv/appdata/beszel/beszel_agent_data | /var/lib/beszel-agent | Agent data |
| /var/run/docker.sock | /var/run/docker.sock | Docker monitoring (read-only) |
